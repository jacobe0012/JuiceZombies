对应接口：领取任务收益 3-2
功能描述：领取对应玩家任务收益
代码逻辑梳理：
第一步：获取缓存配置中的满足领取条件参数
第二步：查出db中对应任务id的玩家数据
第三步：db中数据领取条件参数 > 缓存中领取条件参数
第四步：数据的领取状态是否为已领取
第五步：修改领取状态
第六步：如果是周常日常（发送mq信息给通行证加经验）res = true直接返回 
第七步：领取任务对应资源
第八步：返回资源串初始化对应的任务如果有
public class TaskResult {

    /**
     * 收益
     */
    List<String> reward;

    /**
     * true false
     */
    String res;

    /**
     * 任务id
     */
    Integer activityId;

    /**
     * 阶段
     */
    Integer stage;
 }
